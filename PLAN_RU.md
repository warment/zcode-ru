# План: Русификация ZCode 3.0.1

**Цель:** Добавить русский язык (`ru`) как третью локаль в ZCode с автоматическим выбором в меню языка. Результат — GitHub-репозиторий с патчем и инструкциями.

---

## Этап 1 — Извлечение словарей и анализ кода

**Задание агенту:**
1. Извлечь словари `at` (zh-CN) и `Ze` (en-US) из JS-бандла в JSON
2. Найти и проанализировать код меню выбора языка (где hardcoded `zh-CN`, `en-US`, `system`)
3. Определить количество ключей, структуру, наличие валидации локалей

**Файлы:**
- Бандл: `/tmp/zcode_extracted/out/renderer/assets/index-*.js`
- Результат: `original/zh-CN.json`, `original/en.json`, `analysis.md`

**Результат:** Два JSON-файла + анализ кода меню.

---

## Этап 2 — Перевод словаря (поэтапно)

**Задание агенту:** Создать `ru/ru.json` поэтапно:

**2a.** Перевести критичные элементы (приоритет 1):
- Навигация, меню, кнопки
- Сообщения об ошибках
- Настройки, диалоги

**2b.** Перевести второстепенные элементы (приоритет 2):
- Описания функций
- Подсказки, тултипы
- Тексты помощи

**2c.** Оставить на английском (с пометкой в файле):
- Технические термины: Agent, Plugin, MCP, Goal Mode, Bot Channel
- Названия моделей: GLM-5.2, Claude, GPT
- Строки, которые пользователь вводит руками (промпты)

**Результат:** Файл `ru/ru.json` с разбивкой по приоритетам.

---

## Этап 3 — Патч JS-бандла: добавление ru в nNt

**Задание агенту:**
1. Найти словарь `nNt = {"zh-CN": at, "en-US": Ze}`
2. Добавить третью запись: `"ru": <объект ru.json>`
3. Убедиться что `aNt(e)` корректно обрабатывает `ru` локаль (без изменений, т.к. логика универсальна)

**Результат:** Модифицированный JS-файл с `nNt` содержащим 3 локали.

---

## Этап 4 — Патч меню выбора языка

**Задание агенту:** (на основе анализа из Этапа 1)
1. Найти все места где рендерится меню языка (может быть несколько)
2. Добавить опцию `ru` / `Русский` в список
3. Проверить нет ли валидации допустимых локалей
4. Убедиться что `localStorage["zcode-locale"]` корректно сохраняет `ru`

**Результат:** Модифицированный JS-файл с меню, содержащим 4 опции.

---

## Этап 5 — Сборка и визуальная проверка

**Задание агенту:**
1. Упаковать modified contents обратно в `app.asar`
2. Скопировать `/Applications/ZCode.app` → `/Applications/ZCode-ru.app`
3. Заменить `app.asar` в копии
4. Выполнить ad-hoc подпись: `codesign --force --deep --sign -`
5. Обновить `CFBundleIdentifier` в `Info.plist` на `dev.zcode-ru.app`
6. Запустить ZCode-ru.app
7. **Визуально проверить:** меню языка, отображение русского текста, корректность интерфейса

**Если Gatekeeper блокирует:**
```bash
xattr -d com.apple.quarantine /Applications/ZCode-ru.app
```

**Результат:** Работающее приложение ZCode-ru.app с русским интерфейсом + скриншоты проверки.

---

## Этап 6 — Скрипт повторного применения

**Задание агенту:** Создать `tools/patch.sh`:
1. Принимает путь к ZCode.app как аргумент (по умолчанию `/Applications/ZCode.app`)
2. Автоматически находит JS-файл по паттерну `index-*.js` (не хардкодит имя)
3. Применяет патч: добавляет ru в nNt и меню
4. Пересобирает app.asar
5. Выполняет ad-hoc подпись
6. Проверяет результат

**Результат:** Скрипт `tools/patch.sh` с обработкой ошибок.
